# Technical Specification: Causal Attribution Graph (CAG)
**Version:** 0.1 (Draft)
**Component:** Phase P1 (Capture Layer)

## 1. Schema Definition

The CAG is a Directed Acyclic Graph (DAG) represented as a JSONL stream of nodes and edges.

### 1.1 Node Types
| Type | Attributes | Description |
| :--- | :--- | :--- |
| `SourceOp` | `line_no`, `file`, `op_type` | The original Triton Python operation. |
| `IRNode` | `dialect`, `mnemonic`, `loc_id` | An operation at a specific stage of the MLIR pipeline (TTIR, TTGIR, LLVM). |
| `Pass` | `pass_name`, `timestamp` | A transformation pass in the compiler. |
| `Parameter` | `name`, `value`, `scope` | A tuning knob (e.g., `num_warps`, `BLOCK_M`). |
| `HardwareEvent` | `metric_name`, `value`, `sass_offset` | A profiler-reported hardware symptom. |

### 1.2 Edge Types
| Type | Source -> Target | Description |
| :--- | :--- | :--- |
| `transformed_to` | `IRNode` -> `IRNode` | Tracks an operation across passes. |
| `governed_by` | `Parameter` -> `IRNode` | Indicates which knob influenced this IR generation. |
| `manifested_as` | `IRNode` -> `HardwareEvent` | Links a lowered instruction to a hardware symptom. |
| `derived_from` | `IRNode` -> `SourceOp` | Back-references to the original source. |

## 2. Capture Strategy

### 2.1 Compiler Side (Provenance)
We will leverage Triton's `MLIR_ENABLE_DUMP` and custom logging.
- **Hook:** Intercept `triton.compile` calls.
- **Mechanism:** Capturing the `Attribute` map of the `tt.ModuleOp` which contains `num_warps`, `num_stages`, and `target_arch`.
- **Identity Preservation:** We will use the `FusedLoc` pattern to embed the pass name and iteration index into the IR locations.

### 2.2 Profiler Side (Symptoms)
- **Tool:** `ncu` (Nsight Compute) with `--csv` and `--source-patch`.
- **Join Key:** The `SASS Instruction Offset` mapped to `Line Number` via DWARF line info generated by `ptxas -lineinfo`.

## 3. The "Iteration-Aware" Location Pattern
Standard MLIR location: `loc("myscript.py":10:5)`
CAG-Augmented location: `loc(fused["myscript.py":10:5, "unroll_idx=2", "pass=LoopUnroll"])`

This allows us to disambiguate which specific unrolled iteration is causing a stall.

## 4. Implementation Path (Phase P2)
1.  **`CAG_Capture.py`**: A wrapper around `triton.compile` and `ncu`.
2.  **`MLIR_Parser.py`**: A regex-based (or `mlir-python-bindings` based) parser for IR dumps.
3.  **`Graph_Builder.py`**: Joins the artifacts into the JSONL schema.
